## 什么是搬瓦工官方测速？三个层面说明白

搬瓦工官方下载测速，简单说就是用搬瓦工服务器上托管的文件测试你本地到该机房的网络速度。测的维度有三个：

**延迟（Ping）**：反映网络往返时间。CN2 GIA 线路对国内用户平均延迟在 150-180ms（洛杉矶），香港机房广东方向可以低到 9ms 左右。

**下载速度（Download）**：下载机房提供的测速文件（bin 格式），看本地到机房的实际带宽跑满程度。

**路由追踪（LookingGlass）**：演示站点提供的高级功能，能看到数据包从机房到你 IP 的每一跳走了哪条线路。

这三个一起看才完整。只看延迟不够——延迟低不代表带宽大；只看下载速度也不够——路由走错了晚高峰一样卡。

## 三步测速法：在买之前把机房摸清楚

这是我自己用下来觉得最省事的测速流程，不绕弯子。

**第一步：ping 测试延迟**

在本地打开命令行（Windows 用 cmd，Mac/Linux 直接用终端），输入：


ping 173.242.113.114


把 IP 换成你想测试的机房对应地址。电信用户重点测 DC6 和 DC9，移动用户可以多看看日本大阪软银，联通用户通常 DC6 和荷兰 EUNL_9 都不错。

说实话，光 ping 还不够，因为 ping 走的路由和下载数据走的不一定一样。

**第二步：下载测速文件**

点击表格里对应机房的测速文件链接，直接下载，看下载速度跑到多少。下载完成后那个数字就是你和那个机房之间的实际带宽。25MB 的文件一般几秒内就能测完，够用。

**第三步：演示站点完整测速**

有条件的话，进到对应的演示站点做一次 SpeedTest。这个比下载文件更直观，还有上传速度和延迟抖动数据。

三步走完，基本上对这个机房的情况有底了。

**Q：买之前测速和买了之后实际用速度会不一样吗？**

可能有差异。测速文件走的是文件下载流量，实际使用走的是你的应用协议，路由上可能略有差别。但两者方向一致——测速快，实际使用通常也快；测速差，实际用大概率也差。
